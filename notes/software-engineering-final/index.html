문제 풀이 전후에 순서와 함정만 빠르게 회수하세요.

반드시 외울 순서

  1. SDLC: 요구사항 분석 → 설계 → 구현 → 테스트 → 배포 → 유지보수
  2. 스크럼: 제품 백로그 → 계획 → 스프린트 백로그 → 스프린트 → 리뷰 → 회고
  3. 테스트: 단위 → 통합 → 시스템 → 인수
  4. 기능점수: EI·EO·EQ·ILF·EIF

모델 핵심

  • 폭포수: 단계와 문서가 명확하지만 변경 대응이 어렵다.
  • 프로토타입: 불명확한 요구를 조기 구체화한다.
  • 나선형: 반복마다 위험을 분석한다.
  • 애자일: 짧은 반복과 고객 피드백을 중시한다.

UML 관계 기억법

  • Aggregation: 빈 마름모, 부분 독립
  • Composition: 채운 마름모, 생명주기 공유
  • Generalization: 실선 빈 삼각형
  • Realization: 점선 빈 삼각형

자주 나오는 함정

  • 소프트웨어는 물리적으로 마모되지 않는다.
  • 애자일도 필요한 계획과 문서를 만든다.
  • Review는 제품, Retrospective는 일하는 방식이 중심이다.
  • Validation은 사용자가 원하는 제품인지 확인한다.
